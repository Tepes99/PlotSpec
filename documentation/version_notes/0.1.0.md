# Version 0.1.0

## Summary

`0.1.0` establishes the first working PlotSpec proof of concept. This version turns the project proposition into a runnable application that reads a formal JSON dashboard specification, queries data through DuckDB, renders Plotly components, and serves the result through FastAPI as an embeddable dashboard.

## Scope Delivered

- formal dashboard spec structure with validation
- file-based project discovery from the `projects` directory
- DuckDB-backed query execution for CSV sources
- supported component types: `line`, `bar`, `pie`, `table`
- supported filter types: `single_select`, `multi_select`
- explicit parent-child filter dependency relationships
- FastAPI endpoints for dashboard definition, data, filtered data, and embed rendering
- minimal frontend for filter interaction and Plotly rendering
- Docker and Docker Compose support

## Key Application Behavior

### Specification-driven runtime

Dashboards are no longer handwritten application pages. A dashboard is defined by a JSON file with explicit sections for metadata, data sources, queries, filters, relationships, components, layout, and embed settings.

### Query and filter flow

DuckDB executes the configured queries, and filter state is applied through a narrow service layer. Filter dependencies are resolved in topological order so child filter options react to parent selections.

### Embeddable delivery

The application exposes an iframe-ready dashboard page through FastAPI. The frontend remains intentionally thin and only handles filter state collection, data reloads, and Plotly rendering.

## Included Example

This version ships with one example dashboard:

- `sales-overview`

It demonstrates:

- a CSV-backed data source
- monthly revenue trend visualization
- category revenue breakdown
- filtered detail table
- region-to-category filter dependency

## API Surface

This version exposes the following core routes:

- `GET /health`
- `GET /`
- `GET /dashboards/{dashboard_id}`
- `GET /dashboards/{dashboard_id}/data`
- `POST /dashboards/{dashboard_id}/data`
- `GET /embed/{dashboard_id}`
- `GET /assets/plotly.js`

## Testing and Verification

- filter dependency behavior
- API endpoints

Verified in this version:

- `uv run pytest`
- `docker compose config`
- live FastAPI server startup and endpoint checks

## Known Limitations

This release is intentionally narrow. It does not yet include:

- schema export for external validation tooling
- auth or access control
- database connectors beyond DuckDB-readable local sources
- visual dashboard editing tools
- advanced layout composition
- richer filter types such as date and numeric range filters
- production hardening around query trust boundaries

## Upgrade Notes

This is the initial runnable version of the project. There are no migration requirements yet.
